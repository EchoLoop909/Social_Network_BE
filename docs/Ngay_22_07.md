## 1. AI gợi ý feed bằng Gemini Embedding (Giai đoạn 2)
Cơ chế **content-based + embedding**. AI (Gemini) chỉ làm việc **text → vector**; chọn/xếp/lọc bài là logic Java.

**BE tạo mới:** `EmbeddingService`+Impl (gọi Gemini `gemini-embedding-001`, vector **3072 chiều**), `PostEmbedding` entity + repo, `PostEmbeddingEvent`, `PostEmbeddingService`+Impl, `KafkaEmbeddingListener` (topic `post-embeddings`), `RecommendationService`+Impl, `AiController` (endpoint test/backfill — TẠM).
- **Luồng A (sinh vector):** đăng bài → publish Kafka → consumer ghép `caption + hashtag` → gọi Gemini → lưu `post_embedding`.
- **Luồng B (gợi ý):** `GET /post/recommend` → dựng **vector sở thích** (trung bình CÓ TRỌNG SỐ: save 1.0 / like 0.8 / xem video theo tỉ lệ) → cosine + **min-max** → điểm `0.6 giống + 0.2 mới + 0.2 hot`; user mới (cold-start) = `0.5 mới + 0.5 hot`; loại bài của mình + đã tương tác.
- Sửa `min-max`: khi cosine bằng nhau vẫn xử lý đúng.
- **DB:** bảng `post_embedding` (id_post, vector JSON, model, updated_at) — bảng **#20**.
- Cấu hình: `gemini.api-key/model/endpoint`, `recommend.weight.*` trong `application.properties`.

**FE:** trang chủ `getFeed` giờ gọi **`/post/recommend`** (fallback danh sách thường nếu lỗi).

**Seed dữ liệu:** tạo **16 hashtag** (15 thể loại + `khac`) map **2035 post → mỗi post 1 hashtag**; embed backfill (giới hạn Gemini free 1000 req/ngày).
